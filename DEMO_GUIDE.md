## 🎤 问答准备

### Q1: 如何防止恶意用户垄断画布？

**A**: 我们实现了多重机制：
1. **5 分钟冷却**：放置像素后需要等待 5 分钟
2. **捐款门槛**：需要捐款获得代币
3. **像素覆盖**：其他用户可以覆盖你的像素
4. **保护期**：新放置的像素有 1 小时保护期（可配置）

### Q2: 如何确保捐款真实性？

**A**: 
- **演示模式**：当前使用模拟捐款，所有记录标记为 `isSimulated: true`
- **生产模式**：集成 Every.org webhooks，验证真实支付
- **零资金处理**：我们不接触资金，所有捐款通过 Every.org 直达慈善机构

### Q3: 画布性能如何？

**A**: 
- 使用 Canvas API 高效渲染
- 10,000 像素（100×100）可达 60 FPS
- 实时刷新采用 10 秒轮询（未来可改用 WebSocket）
- 像素数据按需加载

### Q4: 如何扩展到更多项目？

**A**: 
- 数据库设计支持多项目
- 每个项目独立的画布、调色板、代币系统
- 管理员可通过 API 创建新项目
- 支持不同主题、不同慈善机构

### Q5: 成就系统如何工作？

**A**: 
- 数据库已实现成就表
- 4 个预设成就：第一笔、艺术家、慷慨之心、森林守护者
- 解锁成就可获得代币奖励
- 未来可扩展更多成就类型

### Q6: 如何盈利？

**A**: 
- **不盈利**：我们不收取任何费用
- **可选功能**：未来可提供高级功能（如像素保护、NFT 证书）
- **赞助商**：企业可赞助画布项目
- **广告**：可选的非侵入式广告

### Q7: 与 r/place 的区别？

**A**: 
- **公益导向**：捐款支持真实慈善项目
- **持续性**：画布不会重置，永久保存
- **多项目**：支持多个主题画布
- **激励机制**：排行榜、成就系统
- **留言功能**：每个像素可以留言
